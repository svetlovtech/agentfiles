# agentfiles

Sync AI tool configurations to OpenCode from a single source repository.

## Quick Start

```bash
pip install agentfiles

# In a directory with agents/, skills/, commands/, or plugins/:
agentfiles pull                     # interactive pull to all platforms
agentfiles pull --yes               # non-interactive
agentfiles pull --dry-run           # preview only
```

## Core Concepts

**Source repository** — a directory (or git repo) containing `agents/`, `skills/`,
`commands/`, `plugins/` subdirectories. This is the single source of truth.

**Platform** — local OpenCode installation where configs get installed:
- OpenCode (`~/.config/opencode/`)

**Item types** — the four categories agentfiles manages:

| Type | Storage | Platform |
|------|---------|----------|
| Agent | `.md` file or directory | OpenCode |
| Skill | directory with `SKILL.md` | OpenCode |
| Command | `.md` file or directory | OpenCode |
| Plugin | `.ts`/`.yaml`/`.py` file or directory | OpenCode |

## Commands

### Pull (source → platforms)

```bash
agentfiles pull                                    # interactive
agentfiles pull --yes                              # non-interactive
agentfiles pull --target opencode --type agent     # filter
agentfiles pull --only coder,solid-principles      # specific items
agentfiles pull --dry-run --verbose                # preview
agentfiles pull --symlinks                         # symlinks instead of copies
```

### Push (platforms → source)

```bash
agentfiles push                # interactive
agentfiles push --yes          # non-interactive
```

### Sync (bidirectional 3-way merge)

Uses `.agentfiles.state.yaml` checksums for three-way comparison:
- source changed, target unchanged → pull
- source unchanged, target changed → push
- both changed → conflict (manual resolution)

```bash
agentfiles sync
```

### Verify (CI drift detection)

```bash
agentfiles verify                  # exit 0=clean, 1=drift
agentfiles verify --format json
agentfiles verify --quiet          # exit code only
```

CI example:
```yaml
- name: Check config drift
  run: agentfiles verify
```

### Other Commands

| Command | Purpose |
|---------|---------|
| `agentfiles status` | Show installed items per platform |
| `agentfiles list [--tokens]` | List source items (+ token costs) |
| `agentfiles diff` | Compare source vs installed |
| `agentfiles show <name>` | Preview item content |
| `agentfiles adopt` | Import items from platforms into source |
| `agentfiles clean` | Remove orphaned items |
| `agentfiles uninstall` | Remove all installed items |
| `agentfiles init` | Scaffold a new repository |
| `agentfiles update` | git pull + sync in one step |
| `agentfiles branch` | List/switch git branches |
| `agentfiles doctor` | Diagnose environment issues |

## Filter Options

Most commands support:
```bash
--target {opencode,all}
--type {agent,skill,command,plugin,all}
--only item1,item2
--except old-item,deprecated
```

Platform alias: `oc` = opencode.

## Source Repository Structure

```
my-agents/
├── agents/
│   ├── coder.md                   # flat file agent
│   └── debugger/
│       └── debugger.md            # directory agent
├── skills/
│   └── solid-principles/
│       ├── SKILL.md               # required for skills
│       └── references/
├── commands/
│   └── autopilot.md
├── plugins/
│   └── patterns.yaml
├── .agentfiles.yaml               # config (auto-generated by init)
└── .agentfiles.state.yaml         # sync state (auto-managed)
```

## Configuration

File: `.agentfiles.yaml` (searched in CWD then `~/`):

```yaml
default_platforms:
  - opencode
use_symlinks: false
cache_dir: ~/.cache/agentfiles/repos
custom_paths:
  opencode: /custom/path/opencode
```
